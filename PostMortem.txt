Aide a la reprise du projet
- le fichier app.js contient l'initialisation du serveur, avec les routes, les actions a avoir selon les GET/POST du client, et aussi ou lancer le serveur (localhost:1234) ;
- dans le package public se trouve tous les documents dont le serveur peut avoir besoin. Les documents ne sont pas ranges selon l'ordre front-end/back-end mais par type-jeu, ce qui permet de rajouter un jeu relativement facilement :
	- les fichiers bundle_*** que vous pouvez trouver sont generes par gulp, il n'est pas necesssaire de vous en preoccuper ;
	- Chaque page de jeu avec son fichier html + css, des fichiers .js, qui gere la logique.

Comment build ?
Il suffit de faire la commande gulp, qui se charge de tout build et de tout lancer, selon le fichier gulpfile.js
